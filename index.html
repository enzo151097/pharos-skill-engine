Can a Pharos Agent safely execute transactions?

Pharos ExecutionEngine Shield provides agents with a secure transaction sandbox, verifying registry safety, simulation previews, and dynamic gas constraints before value-moving actions are broadcasted on-chain.

Install SDK
safe_execute.js 
const { ExecutionEngineSDK } = require("pharos-execution-engine");

async function runSafeTx() {
  const sdk = new ExecutionEngineSDK(rpcUrl, privateKey, engineAddress);
  
  // Audits target, runs simulation and executes securely

    "0x",                                         // calldata
    "0"                                           // value in ETH
  );
  return receipt;
}

The Problem

Autonomous AI agents perform complex on-chain interactions independently. However, reusing unknown skills introduces critical security vulnerabilities if transactions are not intercepted:

Malicious & Phishing Targets Agents can be tricked by malicious skills into interacting with blacklisted or phishing addresses, draining funds instantly.
Slippage & Frontrunning Losses Transactions executed without pre-flight slippage checks suffer severe price manipulation or sandwich attacks.
Out-Of-Gas Failures Static gas limit configurations cause transactions to revert in congestion periods, wasting gas fees and breaking agent workflows.

The Solution

Pharos ExecutionEngine Shield combines four security primitives into a unified transaction safety layer:

01
On-chain Registry Verification

Instant safety check querying our verified ProtocolRegistry database for whitelist or blacklist address status.

02
Transaction Preview (TxPreview)

Simulates proposed transaction outcomes locally via static call state validations before wasting gas fees on-chain.

03
Slippage Guard Check

Restricts value-slippage deviations within strict maximum thresholds using our gas-efficient SlippageGuard contract.

04
Dynamic Gas & EIP-1559 Optimizer

Queries fee data in real-time to compute optimized Base and Priority fees with a 20% safety limit margin.

How It Works

The SDK intercepts transactions, validates parameters against local and on-chain policies, and executes safely.

  1. Verify Target SDK performs off-chain formatting and checks target code presence.
  2. Registry Scan Queries ProtocolRegistry.checkAddress(target) to block blacklisted addresses.
  3. TxPreview Simulation Executes a static contract call to dry-run outcomes and catch reverts.
  4. Gas & Slippage Guard Validates slippage criteria and estimates EIP-1559 dynamic gas parameters.
  5. Secure Execution Broadcasts the transaction securely via the ExecutionEngine gate.
  6. Receipt Explorer Returns the transaction receipt with a pharosscan link for proof verification.

System Architecture

The Shield acts as an inline proxy between the AI Agent logic and the Pharos Chain ledger.

AI Agent Logic ExecutionEngine SDK ProtocolRegistry (Check) SlippageGuard (Slippage) TxPreview (Simulation) Pharos Chain (Broadcast)
